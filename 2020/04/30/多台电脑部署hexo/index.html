将之前创建的hexo项目文件上传到github上

  • 项目根目录执行命令 git init 则会创建.git文件夹
  • 将以下项目的文件上传到本地仓库,使用git status查看待添加的文件 大致文件如下： git add <文件名> 需要添加的文件名 或者 git add –all将所有文件全部添加 git commit -m “说明” git remote add origin git@github.com:feng8605765/gio-blog.git // 自己的仓库https地址 git pull –rebase origin master git push -u origin master git clone git@github.com:feng8605765/gio-blog.git _config.yml
    package.json
    scaffolds/
    source/
    themes/ npm install hexo-cli -g npm install
    npm install hexo-deployer-git –save // 文章部署到 git 的模块
    （下面为选择安装）
    npm install hexo-generator-feed –save // 建立 RSS 订阅
    npm install hexo-generator-sitemap –save // 建立站点地图 hexo s // localhost:4000 就可以看到了 hexo clean // 清除缓存 网页正常情况下可以忽略此条命令
    hexo g // 生成静态网页
    hexo d // 开始部署 git clone https://github.com/iissnan/hexo-theme-next themes/next
